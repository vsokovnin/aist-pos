# Схема графа сущностей — организация (база) ← люди, проекты

Локальный MD-стор по пути `stores.entities` из конфига. Одна карточка = один файл.

```
<entities>/
  org/<slug>.md       # БАЗОВАЯ сущность
  person/<slug>.md    # affiliation -> org slug
  project/<slug>.md   # org -> org slug
```

**Карточка** = YAML-frontmatter + тело (заметки контекста):

```markdown
---
type: org | person | project
name: "Человекочитаемое имя"
slug: kebab-case-id
aliases: ["другие написания"]
org: <org-slug>          # для person/project: к какой организации (опускается у org)
updated: YYYY-MM-DD
---

Свободные заметки: контекст, договорённости, последние касания.
```

**Правило резолва:** человек ↔ проект связываются ЧЕРЕЗ организацию (общий `org`).
Человек без организации допустим (`org` опущен). При неоднозначности («какой Михаил») —
дизамбигуируй по `org` из контекста разговора. Один файл = один источник правды на сущность;
не плодить дубли — при совпадении `name`/`aliases` обновлять существующую карточку.
