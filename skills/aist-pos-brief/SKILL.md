# AIST POS — утренний бриф

Читает `config/aist-pos.config.yaml`. Коннекторы/вывод — через `config/connectors.md`.

## Когда запускать
- утром / «утренний бриф», «что у меня сегодня». Кандидат на scheduled task (План 3).

## Нативная альтернатива
Если подключён **Productivity** или **Enterprise search** (`/digest`) — утренняя сводка собирается нативно. Этот навык — fallback и когда нужен ваш формат брифа (календарь + почта + обязательства из вашего стора).

## Поток
1. **Календарь сегодня** (`connectors.calendar`): события на текущий день (время, кто, тема).
2. **Важные письма** (`connectors.email`): непрочитанное/требующее ответа за последние сутки — топ по важности (от ключевых людей графа, прямые вопросы). Не весь инбокс.
3. **Открытые обязательства:** прочитай `stores.tasks` (вывод US1) — что висит на мне и что жду от других (с просроченными первыми).
4. **Собери дайджест** (кратко, сканируется за 30 сек):
   ```
   ☀️ Бриф на <дата>
   📅 Встречи: <список время — тема — кто>
   ✉️ Требуют ответа: <2-4 пункта>
   ⏳ Обязательства: мои <n> · ждут меня <m> (просроченные сверху)
   ```
5. **Доставка:** на `output_surface` (file -> `stores.brief`; telegram -> отправить). Пропущенные коннекторы помечай.

## Правила
- Только чтение источников; единственная запись — `stores.brief` (при file) либо отправка.
- Не дублируй извлечение задач — бери готовое из `stores.tasks`. Пусто/устарело -> подскажи запустить US1.
