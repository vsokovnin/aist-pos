# AIST POS — память о сущностях (граф)

Читает `config/aist-pos.config.yaml` (`stores.entities`) и `config/entities-schema.md`.
Поддерживает локальный MD-граф: организация (база) <- люди, проекты.

## Когда запускать
- «обнови карточки», «запомни этих людей/организации», после разбора переписки/встреч (US1/US4 зовут этот скил для резолва).

## Поток
1. **Источник.** Возьми вход: текст разговора/письма/транскрипт (передан вызывающим скилом или указан пользователем).
2. **Извлеки сущности.** Организации, люди, проекты + связи (кто из какой организации, какой проект чьей организации).
3. **Резолв против стора.** Для каждой — найди карточку в `<entities>/{org,person,project}/` по `name`/`aliases`.
   - есть → обнови тело (новый контекст, дата `updated`), не плоди дубль.
   - нет → создай карточку по схеме (`entities-schema.md`); для person/project проставь `org`.
4. **Дизамбигуация.** Тёзки разруливай по организации из контекста. Неоднозначное без сигнала — НЕ угадывай, оставь вопрос пользователю.
5. **Выход.** Кратко: что создано / обновлено (список slug'ов), что осталось неоднозначным.

## Правила
- Один файл = один источник правды на сущность. Организация — базовая; человек/проект всегда пытайся привязать к org.
- Пиши только в `stores.entities`. Не трогай tasks/brief-сторы.
- Ничего не выдумывай: нет сигнала о связи — оставь поле пустым, не фантазируй org.
