# AIST POS — разбор в задачи и долги

Читает `config/aist-pos.config.yaml`. Источники резолвит через `config/connectors.md`.

## Когда запускать
- «разбери входящие в задачи», «что я должен / кто мне должен», после плотного дня встреч.

## Поток
1. **Окно.** Возьми `horizon_days` из конфига (по умолчанию 7).
2. **Источники** (через connector-map; пропускай `none`/недоступные, помечай пропуск):
   - email (`connectors.email`) — письма за окно;
   - messenger (`connectors.messenger`) — чаты за окно;
   - transcripts (`connectors.transcripts`) — транскрипты встреч за окно.
3. **Извлеки обязательства** двумя корзинами:
   - **мои задачи** (что должен сделать я);
   - **долги мне** (что обещали мне / жду от кого-то).
   Для каждого: суть · кто (резолв через `aist-pos-entities`) · срок (если назван) · источник (откуда взято).
4. **Дедуп.** Одно обязательство из нескольких источников — один пункт.
5. **Выход.** Запиши в `stores.tasks` структурой:
   ```
   ## Мои задачи
   - [ ] <суть> — <кому/по чему> — срок <...> — src: <источник>
   ## Долги мне
   - [ ] <кто> должен <что> — срок <...> — src: <источник>
   ```
   Плюс краткое саммари в ответ (сколько задач / долгов, что неоднозначно).

## Правила
- Не выдумывай обязательства — только явные из источников. Сомнительное помечай «(?)», не роняй и не раздувай.
- Имена людей резолвь через граф (US5); неизвестных предлагай завести.
- Пиши только в `stores.tasks`. output_surface не трогаешь (это US2).
