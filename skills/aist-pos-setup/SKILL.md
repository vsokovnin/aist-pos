# AIST POS — установка (setup)

Yar-style онбординг: из нуля до «уверенного L1» базы, поверх которой работают остальные навыки.
Читает `config/aist-pos.config.example.yaml`, `config/connectors.md`, `docs/recommended-setup.md`, шаблоны из `setup-templates/`.

## Когда запускать
- первый запуск после `git clone`; «настрой AIST POS», «/setup», «донастрой».

## Поток
1. **Интервью** (по одному вопросу, разговорно, на «вы»):
   - имя · роль · таймзона;
   - почта: gmail / outlook / нет — **базовый коннектор (рекомендуется)**;
   - календарь: gmail / outlook / нет — **базовый коннектор (рекомендуется)**;
   - мессенджер: telegram / нет;
   - транскрипты: granola / plaud / локально / нет;
   - куда присылать результат: в файл / в telegram (internal: output_surface).
   Почта и календарь — рекомендованный baseline руководителя: без них не работают утренний бриф, подготовка к встрече и бронь календаря. Если их нет — отметьте и предупредите о следствиях, но не блокируйте.
2. **Конфиг.** Скопируй `config/aist-pos.config.example.yaml` → `aist-pos.config.yaml` в рабочей папке пользователя и подставь ответы. Покажи итог, попроси подтвердить.
3. **Детект и health-check коннекторов.** Для каждого заявленного коннектора проверь фактическую доступность в среде:
   - Claude Code — есть ли соответствующий MCP (по `config/connectors.md`); Claude Cowork — подключён ли Connector (проверь/уточни).
   - Статус по каждому: ✅ подключён · ⚠️ заявлен, но не подключён / не авторизован · ➖ не используется.
   - Выдай точный персональный чек-лист «подключить осталось» со ссылками на `config/connectors.md` и `docs/recommended-setup.md`. Если baseline (почта/календарь) не подключён — пометь приоритетом.
   Подключить за пользователя нельзя (OAuth/установка плагина — интерактивный шаг в Claude); навык только детектит и ведёт.
4. **База (9 Must плумбинг)** из `setup-templates/` — создай в рабочей папке, НЕ перезаписывая существующее:
   - корневой хаб-файл (`CLAUDE.md` или `AGENTS.md`) из `root-file.template.md` (подставь {{NAME}}/{{ROLE}}/{{TIMEZONE}}) — root_file + context_seed;
   - `naming-convention.md` из шаблона — naming;
   - `.gitignore` из `gitignore.template` + `git init` (если репо нет) + подсказка про авто-бэкап (3-2-1) — git;
   - сторы: `entities/{org,person,project}/` и `runtime/` — graph + capture sink (схема в `config/entities-schema.md`);
   - `source-map.md` из шаблона — source_map;
   - `memory.md` из шаблона — memory.
5. **Резюме.** Что создано/настроено; статус коннекторов из health-check; чего не хватает — особенно baseline.
6. **Дальше.** Предложи запустить `aist-pos-assess` (оценка ИИ-зрелости) → план развития.

## Правила
- Идемпотентность: существующие файлы НЕ перезаписывать; предлагать merge/skip.
- Ничего вовне; только локальная рабочая папка пользователя.
- Портативность: ноль ссылок на чужую инфру; всё — из шаблонов и конфига репо.
