# AIST POS — бронь календаря

Читает `config/aist-pos.config.yaml`. Календарь — через `config/connectors.md` (нужен WRITE).

## Когда запускать
- «поставь это в календарь», «забронируй время на <задачу>», из договорённостей встречи.

## Поток
1. **Вход.** Задачи/договорённости: из `stores.tasks` (US1) или явно от пользователя.
2. **Кандидаты.** Для каждого: тип (встреча с кем / рабочий блок), длительность, желаемое окно/срок.
3. **Свободные слоты.** Прочитай календарь (`connectors.calendar`) на горизонте; найди свободные окна с учётом timezone из конфига.
4. **Подтверждение (ОБЯЗАТЕЛЬНО перед записью).** Покажи предлагаемые события (что/когда/кто) и спроси подтверждение — запись в календарь необратима/видна вовне.
5. **Запись.** После «да» — создай события через write-коннектор. Верни список созданного со ссылками/id.

## Правила
- НИКОГДА не пиши в календарь без явного подтверждения пользователя (blast-radius: outward-facing).
- Нет write-доступа/коннектора -> не падай: выдай готовый список событий текстом для ручной постановки.
- Не двигай/не удаляй существующие события без отдельного явного запроса.
