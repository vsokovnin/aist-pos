# Capability-модель AIST POS

Движок оценки: 9 базовых способностей («9 Must») по шкале зрелости L1–L5, сгруппированных в 3 кластера. Поверхность простая (3 кластера + стадия), мотор глубокий (per-cap уровни).

## 3 кластера / 9 Must
- **Контекст — агент тебя знает и помнит:** память между сессиями · контекст на запуск.
- **Файлы, структура, доступ:** конвенция имён · корневой хаб-файл · git-версии+бэкап · карта источников+security · интеграции (коннекторы).
- **Захват и граф знаний:** захват входящих · граф сущностей.

## Шкала L1–L5 (по каждой способности)
- **L1** — делаю руками / каждый раз заново;
- **L2** — копится без структуры;
- **L3** — описано и применяется единообразно (целевой базовый уровень);
- **L4** — измеряю и правлю по данным;
- **L5** — система сама себя поддерживает.

## 5 стадий
1. **Болтает с ботом** — AI как чат; система не накапливает.
2. **Помощник** — помогает по задачам, но память/связи слабые.
3. **Оператор** — оперирует на твоём контексте единообразно.
4. **Дирижёр** — процессы измеряются и правятся данными.
5. **Архитектор** — система сама совершенствуется.

## Как считается
- Уровень кластера = среднее по его способностям.
- **Заголовок-стадия** = round(среднее по кластерам) — мотивирующий ориентир.
- **🔻 Слабое звено** = кластер с минимальным средним — показывается отдельно как критический путь.

Гибрид: заголовок мотивирует, слабое звено держит диагноз честным.
